# 论文评审工具 (Thesis Reviewer)

## 评审原则

1. **论文是你自己的作品** - 检查是否有高度模仿他人图片/公式/表格的问题
2. **科学问题要清晰** - 检查是否明确提出了属于作者自己的小问题
3. **逻辑链条要贯穿** - 章节间、段落间、句子间都要有逻辑衔接
4. **格式要统一规范** - 全文格式、标点、用词保持一致

## 评审流程

### 1. 整体结构审查

读取论文全部章节，检查：
- 各章节是否有承上启下的段落
- 章节之间的逻辑链条是否清晰
- 科学问题是否贯穿全文

### 2. 各章节审查

**第一章（引言）**
- 是否有清晰的科学/技术问题陈述
- 国内外现状是否有评述而非简单罗列
- 现状之后是否总结出存在的问题
- 是否有图描述研究内容之间的关系

**第二章（理论基础）**
- 章节名称是否规范
- 别人的工作与自己的工作是否分开

**后续章节**
- 每章开头是否总结前文、引出本章
- 每章结尾是否有承上启下段落
- 公式、图的引用是否规范

### 3. 格式检查

详细检查清单见 [references/checklist.md](references/checklist.md)

### 4. 输出评审报告

按优先级分类输出问题：
- **P0 致命问题**: 科学问题不清、逻辑断裂、抄袭风险
- **P1 重要问题**: 格式不一致、引用不规范、图表问题
- **P2 建议优化**: 表述优化、语言润色

## 使用方法

对论文项目进行全面评审：
```
读取所有.tex章节文件
对照评审清单逐项检查
输出结构化评审报告
```

评审报告格式：
```markdown
# 论文评审报告

## 一、整体评价
[论文的整体质量评价]

## 二、P0 致命问题
| # | 问题 | 位置 | 具体描述 |
|---|------|------|----------|
| 1 | ... | ... | ... |

## 三、P1 重要问题
...

## 四、P2 建议优化
...

## 五、修改优先级建议
1. 首先解决 P0 问题
2. 其次处理 P1 问题
3. 最后优化 P2 细节
```
